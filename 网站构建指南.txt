# 给 Codex 的构建说明：沉浸式博客网站（Astro + Markdown/MDX + GitHub Pages）

请根据以下完整规格，自动生成一个可运行、可部署、可持续写作维护的个人博客项目。

## 1. 项目目标

构建一个 **沉浸式博客网站**，满足以下要求：

* 使用 **Git 仓库**管理内容与运维
* 使用 **Astro** 作为框架
* 使用 **Markdown / MDX** 作为内容格式
* 使用 **GitHub Pages + GitHub Actions** 自动部署
* 使用 **Giscus** 作为评论系统
* 暂时 **不接入搜索**，但代码结构要预留后续接入 **Pagefind** 的空间
* 图片采用 **本地管理**，统一放在 `src/assets` 或 `public/images`
* 风格上强调 **沉浸式阅读体验**，不是文档站，不是传统技术博客列表站

## 2. 技术路线

请按以下技术路线构建：

* 框架：Astro
* 内容：Markdown / MDX
* 部署：GitHub Pages + GitHub Actions
* 评论：Giscus（基于 GitHub Discussions）
* 搜索：暂不实现，但保留后续扩展结构
* 图片：封面图本地管理，统一放在 `src/assets` 或 `public/images`

## 3. 核心设计方向

网站整体风格采用：**夜航型沉浸阅读**

关键词：

* 深夜感
* 冷静
* 专注
* 未来感轻一点，不要赛博朋克过头
* 大留白
* 大字号标题
* 首屏只有一句话和入口
* 滚动时有微弱视差和渐变
* 卡片边缘弱发光
* 图片尽量少而大

适合内容类型：

* 思考
* 技术随笔
* 设计观察
* 生活叙事

## 5. 字体使用要求

请将字体系统纳入整体视觉规范，并统一为可维护的 design tokens / CSS 变量。字体选择需服务于“夜航型沉浸阅读”的整体气质：克制、清晰、偏科技深度，但不要过度赛博化。

本项目请使用以下字体组合：

* 中文：`Source Han Sans SC`
* 英文标题 / UI：`Geist Sans`
* 英文正文：`Inter`
* 代码：`Geist Mono`

### 5.1 字体使用原则

* 中文正文优先保证稳定、克制、清晰的屏幕阅读体验
* 英文标题与导航需要具备适度现代感与科技感，但不能显得激进或过强未来风
* 英文正文要保证长时间阅读的舒适度
* 代码字体要与正文风格协调，保持清晰、安静、精确
* 全站尽量控制字体家族数量，避免堆砌和风格割裂

### 5.2 推荐层级

请按以下层级使用字体：

* 中文正文 / 中文 UI：`Source Han Sans SC`
* 英文 Hero 标题 / 英文导航 / 英文栏目标题：`Geist Sans`
* 英文正文 / 英文说明文字 / 英文 meta 信息：`Inter`
* 代码块 / 行内代码 / 命令行片段：`Geist Mono`

### 5.3 工程要求

请在工程中统一封装字体变量，例如：

* `--font-sans-zh`
* `--font-sans-en-display`
* `--font-sans-en-body`
* `--font-mono`

要求：

* 优先采用可自托管方式接入字体
* 尽量优先使用 variable font 版本（如可行）
* 在全局样式中统一声明字体栈
* 为系统字体设置合理回退
* 控制字体资源体积，避免对首屏加载造成明显负担

### 5.4 字体风格约束

请避免使用以下类型字体作为主字体：

* 过强赛博朋克风
* 过度机械感或强装饰性字体
* 不适合长时间阅读的展示型字体

整体效果应体现：

* 安静
* 清晰
* 现代
* 有轻微科技感
* 适合深色沉浸式阅读

## 6. 设计原则

请务必遵守以下原则：

### 5.1 首页不是文章列表页

首页应是一个 **氛围入口页**，而不是普通博客流。

首屏应包含：

* 一句简洁但有气质的站点介绍语
* 轻量背景氛围（渐变、弱噪点、弱光晕、轻微视差皆可）
* 少量主要入口，例如：

  * Blog
  * Essays
  * Notes

首页下半部分可以展示：

* 精选文章
* 最近文章
* 一个简短的 About 区块

但整体信息密度必须克制。

### 5.2 阅读体验优先

文章页要把阅读体验放在第一位。

要求：

* 正文宽度控制在约 `680px ~ 760px`
* 行高舒适（建议约 `1.8`）
* 段落、标题、引用、图片说明层级清晰
* 代码块样式要精致但不过度抢眼
* 文章头图区克制，不要做花哨的大 Banner

### 5.3 少组件堆砌，多空间感

不要做成组件展示站。

避免：

* 过多标签
* 过多元信息
* 过多图标
* 过度动画
* 瀑布流式信息轰炸

应优先追求：

* 节奏感
* 空间感
* 安静感
* 品质感

### 5.4 动效只做弱反馈

建议只实现轻微动效：

* hover 轻微上浮
* 页面淡入
* 元素渐显
* 背景弱渐变流动
* 非强制的轻微滚动视差

不要加入炫技型重动画。

## 9. 页面具体要求

### 9.1 首页 `/`

首页应包含：

Hero 文案请先使用占位内容，但要保留易替换结构。

### 9.2 列表页

* 顶部简短说明
* 内容列表
* 每个内容卡片显示：标题、描述、日期、标签

### 9.3 文章详情页

文章页应包含：
